## 一、5 类系统性问题（根因）

### 模式 1 · 前后端契约靠 Mock 掩盖，真机即断
| 问题 | 修复 |
|---|---|
| 发布 `quantity`↔后端 `totalWeight`、`pricing{}`嵌套↔扁平、`scatter`↔`bulk` | 前端提交扁平化对齐；出货值 `scatter→bulk`、定价 `market→spot` |
| 订单 `OrderStatus` 无 `selling`（销售中恒空）、`counterparty.address` 后端不返回 | `list('selling')` 走挂单分支、`detail` 返回对手方 address；selling 详情用挂单 id 回退查询 |
| `/lock/buyer-limit`、`/relay/apply`、`/relay/step` 无前端调用方 | api 层补 `getBuyerLimit`/`applyRelay`/`relayConsent`/`updateRelayStep`，页面接线 |

### 模式 2 · 后端不落地业务规则（用常量/占位/硬编码）
| 问题 | 修复 |
|---|---|
| **保证金按 `金价×10%`≈¥103/g（违反 C1 固定 ¥10/g，偏差~10 倍）；可交易额度写死常量** | 新增 `MARGIN_UNIT_FEN`（金 1000/银 50/铂 500 分/克）；锁价冻结、可购买上限、可交易额度三处统一按固定单价；`weight×单价` 确定性重算，冻结/解冻精确 |
| 违约降级恒 `-1`、扣罚写死 ¥1、无限制天数 | `recordDefault` 按 E1 阶梯（1/2/3/4+ → 3/7/15/30 天 + 降 1/2/3/清零至 L1）；E2 扣款=克重×单价并赔付对手方（`margin.compensate`） |
| 后端未兜前端规则 | 补：地址≤5+类型校验、充值≥¥500、实名通过后锁定、发布前置联系方式、仲裁材料 1~5 张+≤500 字校验并落库、订阅≤8/品类、退款 C4 守卫 |
| 订单号随机/9999 硬编码 | B9：母单 16 位（挂单派生，子单共享）+ 子单 2 位顺序 = 18 位 |
| 等级显示 L0、费率表仅 5 行 | User.level 默认 L1；费率表 L1–L9；升级进度按每 10 笔算 |

### 模式 3 · 定时任务违反已定稿 A2/B3（会误罚守约方）
- 撤除「4h/仲裁到期自动判双方/申请方违约」；改为 A2 继续计时（仅记录逾期观测）、B3 人工判定。
- 新增 **B2**：一方确认 + 另一方超时 24h → 自动完成并释放双方保证金。

### 模式 4 · 交割闭环断裂
- 平台代交接：前端「支付¥100并申请」空按钮 → 真调 `applyRelay`；**B6 需对方同意**（新 `POST /orders/:no/relay/consent`）；**B7 末步完成 → 释放双方保证金 + 双方成交数+1**。
- 卖家侧保证金：锁价时同额冻结（尽力而为，解冻端 clamp 安全）。

### 模式 5 · 设计稿合规/软约束交互大面积未还原
- 锁价：确认前静默校验联系方式跳补全（pin7）；确认弹窗补 A7 不可取消警示 + 4 条必读 +「我已阅读」勾选门槛；可购买上限卡（级别/保证金/超限/补足保证金链接）；失败页可达。
- 发布：商品名称必填输入 + 确认发布清单（屏7）+ 须知弹窗改居中不可遮罩关闭 + 结果页富信息。
- 我的：违约处罚阶梯表 + 判定弹窗 + 功能受限 banner + 24h 申诉窗口 + 退款页/tab + 充值克重试算 + mine 菜单结构/右侧值/地址入口。
- 首页：品类折页 Tab + 排序栏 + 上拉分页 + 结果计数 + 大盘卡随品类换色 + 订阅底部弹窗 + 散出标签配色。
