# 微金100 · 微信小程序

贵金属（黄金 / 白银 / 铂金）**C2C 撮合 + 线下交割**平台的微信小程序项目。前后端分离，后端由他方负责，前端对接接口文档。

> 游客可浏览全部行情；会员可操作（锁价 / 发布）。核心交易：商家发布现货挂单 → 买家锁价 → 4 小时内线下实名交割 → 双方确认完成。

## 当前阶段

**前后端主体功能已实现并全链路测试通过。** UI 设计稿全部完成 → 接口契约定稿 → 业务规则定稿（48 条）→ 前端（原生小程序 + TS，23 页 / 8 组件）+ 后端（NestJS + PostgreSQL/Prisma + 脉动行情真链路）全部落地 → **对照设计稿全面排查并修复**（漏做/多做/契约不符，见 [设计稿对照排查报告_2026-07.md](设计稿对照排查报告_2026-07.md)）→ **建立五层测试体系 + 真库真机联调**（见下）。剩余为合规资质与第三方接入（支付/实名 OCR/上架），详见 [HANDOFF.md](HANDOFF.md) §6。接手请先读 **[HANDOFF.md](HANDOFF.md)**。

## 测试与联调（自带内嵌 PostgreSQL，免装库）

五层测试，全部真实、可复现：

| 层 | 断言数 | 命令 | 说明 |
|---|---|---|---|
| 后端单元 | 40 | `cd backend && npm run test:unit` | 保证金数学 / 状态机 / 出货约束 / 裁决 / 信用修复（fake 依赖，无需库） |
| API 集成 | 68 | `npm run test:api`（需服务+库） | 全 43 路由 + 业务流 + 负向 + 管理端裁决 |
| **全业务场景** | 70 | `npm run test:scenarios`（需服务+库） | 按业务规则 A–H 逐条建场景（正向+负向+边界） |
| 前端契约 harness | 18 | `cd frontend && npm run test:integration` | 前端**真实** api/auth/request 代码（wx 垫片）→ 真后端，**无 GUI** |
| 真机 e2e | 9 | `cd frontend && npm run test:e2e` | 真小程序运行时（微信开发者工具自动化）→ 真后端 |

**一键跑服务端三套（156 断言，自带内嵌 PG，端口自动清理）：**
```bash
cd backend && npm run test:integration
```
**本地起真库联调**（供微信开发者工具连本地后端）：
```bash
cd backend && npm run dev:pg      # 内嵌 PostgreSQL + 迁移 + 种子（含 devuser001）
cd backend && npm run start       # 起后端（127.0.0.1:3100）
# 微信开发者工具打开 frontend/ 目录即可（env 已 USE_MOCK=false）
```
> 真机 e2e 前提：开发者工具登录 + 安全设置开「服务端口」「自动化默认信任项目」。DevTools **无无头模式**，自动化必开 GUI 窗口（跑完自动关）。

## 查看设计稿

设计稿为自包含 HTML，直接用浏览器打开即可。右上角开关：

- **设计说明（S 键）**：显示每屏的设计规则批注。
- **批注模式（A 键）**：点击屏幕任意位置添加修改意见，可导出 Markdown。

## 技术栈（拟定）

- 前端：微信小程序，**原生 + TypeScript + 自建组件库**（已定）；手机号微信一键授权
- 后端：RESTful + WebSocket（行情推送），由他方提供
- 鉴权：微信登录 + JWT；游客态浏览 + 操作时静默校验
