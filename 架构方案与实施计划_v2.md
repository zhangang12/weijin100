# 微金100 · 架构方案与实施计划 v2

> 在 [v1](技术方案与开发计划_v1.md)（技术栈/联调约定/业务路线）基础上，**定稿部署架构（单 ECS 起步）+ 可扩展设计 + 模块级实施计划**。
> 核心原则：**单机起步省成本，但所有外部依赖走驱动抽象、应用无状态——将来"加机器/换托管/上云存储"都是改配置，不重构。**
> 配套：`技术方案与开发计划_v1.md`、`backend/prisma/schema.prisma`、`资源准备清单.md`。日期 2026-06-28。

## 1. 部署架构（单 ECS 起步）

```
微信小程序(原生+TS)
   │  HTTPS / WSS
   ▼
┌─────────────────── 一台 ECS（Docker Compose）───────────────────┐
│  Nginx  ──►  NestJS 后端(Node)                                  │
│   ├─ /api  反向代理          ├─ PostgreSQL  (容器, 数据在 /data) │
│   ├─ /ws   WebSocket 转发     ├─ Redis       (Sprint3+, 容器)    │
│   └─ /uploads 静态(仅商品图)  └─ 本地磁盘 /data/uploads          │
└────────────────────────────────────────────────────────────────┘
   │ 调用外部
   ├─ 脉动行情 PulseData (HTTP+WS)
   ├─ 微信开放平台 (登录 jscode2session / 支付V3 / 订阅消息)
   ├─ 阿里云短信   ├─ 实名核身(OCR+人脸)
   │
   └─ 数据盘 /data：PG 数据 + uploads；cron 定时备份(pg_dump + rsync → 磁盘快照/异地)
```

- **证件/实名照**：不走 Nginx 静态，走**后端鉴权下载接口**（校验登录+本人）；商品图走 Nginx 静态直出。
- 规格起步 **2核4G**（建议 4核8G）；系统盘 + **单独数据盘 `/data`**（PG 数据 + uploads，便于快照/扩容）。

## 2. 后端分层与模块（NestJS）

| 层 | 内容 |
|---|---|
| `common` | 响应信封拦截器、异常过滤器、JWT 守卫、装饰器、校验管道 |
| `config` | 环境配置 |
| `prisma` | PrismaModule/Service（数据访问） |
| **`infra`（可扩展核心，全部带驱动抽象）** | `storage`(本地/OSS) · `cache`(Redis/内存) · `queue`(BullMQ/即时) · `wechat`(登录/支付) · `sms`(mock/阿里云) · `market`(脉动/可换源) |
| `modules`（业务） | `auth` · `user` · `margin` · `level` · `listing` · `lock` · `order` · `default` · `alert` · `address` · `market`(读) · `admin`(预留) |

## 3. 可扩展性设计（"后面能扩展"的关键）

> 原则：**外部依赖一律 interface + driver；状态外置（DB/Redis）；应用无状态。** 切换只改配置/加驱动，不动业务代码。

| 扩展点 | 现在 | 将来 | 怎么切 |
|---|---|---|---|
| **文件存储** | 本地磁盘 `/data/uploads` | OSS + CDN | 实现 `OssDriver`，改 `STORAGE_DRIVER=oss` |
| **数据库** | ECS 自管 PostgreSQL | 阿里云 RDS | 改 `DATABASE_URL`（Prisma 无感） |
| **缓存/锁/队列** | Redis 同机（Sprint3）/ dev 用内存降级 | 云数据库 Redis/Tair | 改 `REDIS_URL` |
| **行情源** | 脉动 PulseData | 换/加数据源 | 实现新 `QuoteProvider` |
| **登录/支付** | 微信 dev 测试号 | 真号 + 真商户号 | 改 `WX_*`/`WXPAY_*` 配置 |
| **短信** | mock 驱动 | 阿里云短信 | 实现 `AliSmsDriver` |
| **应用实例** | 单实例 | 多实例 + SLB | 无状态(JWT+状态在DB/Redis)，直接水平扩展 |
| **多品牌/租户** | 单租户 | 预留 | 模型可加 `tenantId`（暂不启用） |

| Sprint | 后端模块任务 | 前端 |
|---|---|---|
| **0 地基** | ✅ NestJS 骨架(信封/异常/前缀/CORS) · ✅ Prisma+16表+本机 PG 迁移 · ✅ 行情真链路 · ⬜ **infra 抽象骨架**(storage-local / cache(redis\|memory) / queue(bullmq\|immediate) / wechat / sms-mock) · ⬜ 种子数据 · ⬜ `docker-compose.prod`+Nginx+备份脚本 | 接真后端行情、env 切换 |

## 5. 部署方案（单 ECS）

- **编排**：`docker-compose.prod.yml` = nginx + backend + postgres(+redis Sprint3)；env 注入（`.env` 不入库）。
- **Nginx**：HTTPS(证书) + 反代 `/api` + WSS `/ws` + 静态 `/uploads`(仅商品图)。
- **备份（不可省）**：cron → `pg_dump` + `rsync /data/uploads` → 磁盘快照 + 异地，保留 N 天。
- **进程**：容器 `restart: always`；后端无状态，可随时重启/扩容。
- **演进触发**：真实资金流动/规模上来 → **DB 先迁 RDS**；并发高 → 加实例 + SLB + 云 Redis；存储增长 → 上 OSS。

## 6. 中间件清单

| 中间件 | 现在 | 说明 |
|---|---|---|
| PostgreSQL | ✅ 必需（已跑） | 主业务库；ECS 自管 → 将来 RDS |
| Redis | Sprint 3 起 | 锁/幂等/队列/缓存；dev 可内存降级 |
| Nginx | 部署期 | HTTPS/反代/WS |
| ~~OSS~~ | ❌ 取消 | 改本地 `/data/uploads`（保留 Storage 抽象，可后接） |
| 外部服务 | 按 Sprint | 脉动行情 / 微信(登录·支付·订阅) / 短信 / 实名核身 |

## 7. 风险与演进触发

- **单 ECS 单点** + **自管 PG 必须定时备份**（金融数据，最高优先）。
- **证件合规**：鉴权下载、加密存储、不公开 URL。
- **行情单点**：缓存 + 断流降级(用最后快照 + 告警)。
- **微信支付/资质周期长**：早启动。
- 触发拆分托管：资金真实流动 / DAU 上升 / 容量上升。

## 8. 决策记录（追加 2026-06-28）

- 部署 = **单 ECS** 起步（docker-compose: nginx+nestjs+postgres[+redis]）；**文件本地落盘 `/data/uploads`，取消 OSS**（保留 Storage 驱动抽象，可后接）。
- 可扩展靠**驱动抽象 + 无状态应用 + 状态外置**：DB→RDS 改连接串、存储→OSS 加驱动、单实例→多实例直接横扩。
- Redis 推迟到 Sprint 3；dev 期缓存/队列走内存降级驱动。
