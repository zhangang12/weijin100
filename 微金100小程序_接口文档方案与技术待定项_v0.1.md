# 第一部分：全局约定（Global Conventions）

## 1.1 架构与环境

- 前后端分离，后端提供 RESTful JSON 接口 + WebSocket 行情推送。
- **Base URL**：`https://{env-host}/api/v1`
- **环境**：`dev` / `staging` / `prod` 三套，前端按构建环境切换 baseURL。
- 所有请求/响应 `Content-Type: application/json; charset=utf-8`（文件上传除外）。

## 1.2 鉴权方案（微信登录 + JWT）

```
小程序启动 → wx.login() 拿 js_code
          → POST /auth/login { code }
          → 后端 jscode2session 换 openid/unionid/session_key
          → 签发 JWT：accessToken(短期) + refreshToken(长期)
前端存储 token → 后续请求头注入：Authorization: Bearer {accessToken}
accessToken 过期(401) → 用 refreshToken 换新 → 失败则重新 wx.login
```

- **游客态**：浏览类接口（行情、挂单列表、挂单详情、金价等）**免登录**可访问；可下发「游客 token」或直接匿名放行（🔶待确认）。
- **会员操作态**：锁价、发布、充值、订单操作等**必须鉴权**。
- **静默校验（核心交互规则）**：前端在用户点「锁价 / 发布」时，先查 `GET /me/eligibility` 或主页已返回的资质标志：
  - 缺**联系方式 / 实名 / 保证金** → **直接路由跳「我的」对应补全页**（无弹窗），补完原路返回并保留上下文（选中货品、数量）。
  - 后端在**提交时仍需二次硬校验**（前端跳转只是体验优化，不能作为安全边界）。
- 请求头约定：

| Header | 说明 |
|---|---|
| `Authorization: Bearer {token}` | 鉴权（游客可空） |
| `X-Client: miniapp` / `X-Client-Version` | 客户端与版本 |
| `Idempotency-Key: {uuid}` | 幂等键（仅写操作：锁价/发布/充值/确认完成等） |
| `X-Request-Id` | 链路追踪（前端生成，后端回传） |

## 1.3 统一响应结构

```jsonc
{
  "code": 0,                 // 0=成功；非0=失败（数值，见错误码总表）
  "bizCode": "OK",           // 语义常量，失败时如 "MARGIN.INSUFFICIENT"
  "message": "成功",          // 可直接 toast 的中文提示
  "data": { /* 业务数据，失败时为 null */ },
  "requestId": "a1b2c3..."   // 链路追踪
}
```

- 业务层统一 **HTTP 200 + 信封 code**；仅传输/网关层错误用真实 HTTP 4xx/5xx。
- 分页响应统一：

```jsonc
"data": {
  "list": [ /* ... */ ],
  "page": 1, "pageSize": 20, "total": 128, "hasMore": true
}
```

## 1.4 金额 / 数字 / 时间 / 单位约定

| 项 | 约定 |
|---|---|
| **金额** | 统一以**「分」整数**传输（如 `¥3,096,630.00` → `309663000`）；或字符串小数（🔶二选一，见待定项 E）。**禁止用 float**。 |
| **单价（元/克）** | 字符串/定点小数，保留 2 位（行情/大盘价、成交价）或 3 位（发布溢价/固定价 `2.500`）。 |
| **克重** | 数值，单位 `g`，保留 2 位小数（锁价后），展示千分位。 |
| **百分比** | 字符串如 `-0.49%` 或数值 `-0.0049`（🔶统一其一）。 |
| **时间** | ISO8601 带时区 `2026-04-28T12:57:40+08:00`；前端按场景格式化（订单用绝对时间，禁用「x分钟前」）。 |
| **涨跌色** | 枚举 `trend: "up" | "down" | "flat"`；**红涨绿跌**（up=红 `#E0492F`，down=绿 `#1FA766`），前端据 trend 上色，不靠正负号判断。 |
| **金额计算口径** | **一律以后端为准**，前端仅做展示与试算；克重×单价、合计、限额、溢价均由后端二次计算。 |

**关键业务常量（来自设计稿，🔶需后端确认为权威值或改为配置下发）：**

| 常量 | 值 |
|---|---|
| 保证金单价 | 黄金 `¥10/g`、白银 `¥0.5/g`、铂金 `¥5/g` |
| 可交易额度 | `额度(g) = 保证金可用余额 ÷ 该金属单价` |
| 最低充值 | `¥500` 起 |
| 退款 | T+1 原路返还、无手续费；需无在途订单且无未结违约 |
| 级别 | L1–L9，**每 10 笔成交升 1 级**（买卖各计 1 笔），运营期免佣 |
| 违约处罚 | 1次=限制3天+降1级；2次=7天+降2级；3次=15天+降3级；4次+=30天+清零至L1；扣款=违约克重×单价 |
| 信用修复 | 每 30 笔无违约消除 1 条最早记录；违约累计=近 12 个月未修复记录数；申诉窗口 24h |
| 履约时限 | 锁价后 **4 小时**内线下交割 |
| 平台代交接 | `¥100/单`，发起方支付（可协商分摊），需对方同意；验货+买家打款即完成并释放双方保证金 |
| 仲裁 | 聊天截图≤5张（必填）、情况说明≤500字（必填）；平台 2h 内致电；提交后暂停 4h 倒计时 |
| 订单号 | 16 位母单（未锁价）；锁价后 +2 位子单号 = 18 位：`YYMMDD(6)+用户尾码(4)+流水(6)+子单(2)` |
| 发布图片 | 1–9 张，单张 ≤10MB，jpg/png，首图封面 |

## 1.5 数据字典 / 枚举（全局复用）

```jsonc
metal:        ["gold", "silver", "platinum"]        // 黄金/白银/铂金
category:     ["board", "bar", "scrap", ...]         // 板料/金条(银条/铂条)/旧料 🔶按品类细化
shipMode:     ["whole_all", "whole_fixed", "bulk"]   // 整出全量/整出固量/散出
payMethod:    ["cash", "transfer"]                   // 现金(红)/转账(蓝)
priceMode:    ["market", "fixed"]                    // 大盘价+溢价 / 一口固定价
trend:        ["up", "down", "flat"]                 // 红涨绿跌
kycStatus:    ["none", "pending", "verified", "rejected"]
level:        "L1" ~ "L9"
lockStatus:   ["pending_confirm", "processing", "success", "failed"]
orderStatus:  ["selling", "locked_pending", "completed",
               "relay_inspecting", "arbitrating", "cancelled", "defaulted"]  // 🔶部分推测
deliveryMethod:["face_to_face", "platform_relay"]    // 当面自交/平台代交接
role:         ["buyer", "seller"]
alertCond:    ["above", "below"]                     // 涨到/跌到
```

> 建议后端提供 `GET /config/dict` 一次性下发全部枚举与展示文案，前端不硬编码中文。

# 第三部分：接口清单（按模块）

> 标注：🔓=游客可访问，🔒=需登录，🔑=需登录+幂等键。

## 0. 鉴权与通用

| 方法 | 路径 | 鉴 | 说明 | 入参 | 出参(data) |
|---|---|---|---|---|---|
| POST | `/auth/login` | 🔓 | 微信登录换 token | `{code}` | `{accessToken, refreshToken, expiresIn, user}` |
| POST | `/auth/refresh` | 🔓 | 刷新 token | `{refreshToken}` | `{accessToken, expiresIn}` |
| POST | `/auth/phone` | 🔒 | 解密微信手机号（可选，预填联系方式） | `{code}` 或 `{encryptedData,iv}` | `{phone}` |
| GET | `/config/dict` | 🔓 | 枚举字典 + 展示文案 | — | 全量枚举 |
| GET | `/config/app` | 🔓 | App 配置（客服、协议链接、版本、最低充值等常量） | — | `{csWechat,csPhone,onlineHours,agreementUrls,minRecharge,...}` |
| POST | `/uploads` | 🔒 | 通用文件上传（图片/证件/截图） | multipart `file` + `bizType` | `{fileId, url}` |
| GET | `/me/eligibility` | 🔒 | 操作前资质聚合（静默校验依据） | `action=lock|publish` | `{realName:bool, contact:bool, marginOk:bool, functionStatus, maxQty}` |

| GET | `/seller/publish/eligibility` | 🔒 | 发布前置校验（实名+联系方式+保证金） | 出：`{realName, contact, marginOk, level, deposit, maxQty, minQty}` |
| GET | `/seller/publish/limit` | 🔒 | 等级×保证金→可发布上限 | 入：`metal` 出：`{level, deposit, minQty, maxQty}` |
| POST | `/seller/publish/calc-price` | 🔒 | 实时参考价试算（可选） | 入：`{metal, priceMode, cash/transfer, floorPrice}` 出：`{refPriceCash, refPriceTransfer}` |
| POST | `/listings` | 🔑 | **发布挂单**（确认发布） | 见下 body；出：`{listingId, status}` |
| GET | `/listings/{id}` | 🔓 | 商品详情（发布成功后查看） | — |

**发布 body（`POST /listings`）：**
```jsonc
{
  "metal": "gold",
  "category": "scrap",                 // 旧料需填"折后实重"
  "goodsName": "品牌板料 / 品牌黄金带证书",
  "quantity": 10000.00,                // g, 必填, ≤可发布上限
  "shipMode": "whole_fixed",           // whole_all|whole_fixed|bulk
  "lotSize": 1000,                     // whole_fixed 时每份 g
  "minBatch": null,                    // bulk 时起批量(≥1, ≤quantity)
  "pricing": {
    "priceMode": "market",             // market|fixed
    "cash":     { "enabled": true, "sign": "+", "premium": "2.500" },
    "transfer": { "enabled": true, "sign": "+", "premium": "3.500" },
    "floorPrice": null,                // 仅 market，可空；fixed 模式不传 sign/floorPrice
    // fixed: "cash":{"enabled":true,"price":"1032.000"}
  },
  "images": ["fileId1(封面)", "fileId2"],   // 1~9 张
  "agreedRules": true                  // 合规5条须勾选
}
```
**校验**：现金/转账至少启用一个；`bulk` 起批量 1g 起且 ≤总量；图片 1–9 张≤10MB；后端二次计算 refPrice。**发布失败返回 `{failReason, suggestion, errorCode}`**（设计稿明确码 `E_NOT_SPOT_4021` 非水贝现货）。

## 4. 买家锁价 (lock)

| 方法 | 路径 | 鉴 | 说明 | 入参 / 出参 |
|---|---|---|---|---|
| GET | `/market/listings/{id}` | 🔓 | 锁价主页详情（复用行情详情） | 货品+卖家+出货方式+库存+收款支持+初始快照 |
| GET | `/lock/quote/{listingId}` | 🔓 | 实时报价快照（高频） | 出：`PriceSnapshot`（含 snapshotVersion） |
| GET | `/lock/buyer-limit` | 🔒 | 我的购买上限（等级+保证金） | 入：`listingId` 出：`{buyerLevel, deposit, maxBuyableQty, overLimit}` |
| POST | `/lock/orders` | 🔑 | **提交锁价** | 见下；出：`{lockOrderId, status:"processing"}` |
| GET | `/lock/orders/{id}` | 🔒 | 查询锁价处理结果（轮询/或 WS） | 出：`{status, failReason?}` |
| GET | `/lock/orders/{id}/seller-contact` | 🔒 | 锁价成功后取卖家联系方式 | 出：`{phone, wechat, deliveryStatus}` |

**提交锁价 body（`POST /lock/orders`）：**
```jsonc
{
  "listingId": "L_88001",
  "shipMode": "whole_fixed",
  "qty": 3000.00,            // 或 portions:3（整出固量）
  "payMethod": "cash",
  "snapshotVersion": "v-1714280260",   // 价格快照版本，后端校验时效
  "agreedRules": true
}
```
**服务端校验**：① 快照未过期（否则 `LOCK.QUOTE_EXPIRED`）；② 库存充足且未被抢锁（`LOCK.GRABBED`/`STOCK_INSUFFICIENT`）；③ 联系方式已补全（`CONTACT_REQUIRED`，前端已拦截）；④ 幂等键防重复提交；⑤ 保证金/上限二次校验（软约束，但提交时建议硬校验，🔶见待定项 D）。提交后进入**「处理中」异步缓冲**（并发仲裁窗口），再异步出成功/失败。

## 5. 我的订单 / 交割 (orders)

| 方法 | 路径 | 鉴 | 说明 | 入参 / 出参 |
|---|---|---|---|---|
| GET | `/orders` | 🔒 | 订单列表（分板块+筛选+搜索） | `tab=selling|locked|done, filter=all|sell|buy, orderNo?, page` |
| GET | `/orders/badge` | 🔒 | 未完成数角标 | `{pendingCount}` |
| GET | `/orders/{orderNo}` | 🔒 | 订单详情（仅当事人） | `Order` 全字段 + 倒计时 |
| POST | `/orders/{orderNo}/cancel` | 🔒 | 取消（仅卖家、仅剩余库存、未锁价部分） | — |
| PUT | `/orders/{orderNo}` | 🔒 | 修改挂单参数（仅卖家、未锁价） | g重/报价/出货方式 |
| POST | `/orders/{orderNo}/confirm-complete` | 🔑 | 确认交易完成（买卖各一次，双方确认才终结） | 出：`{myConfirmed, peerConfirmed, status}` |
| POST | `/orders/{orderNo}/arbitration` | 🔑 | 提交仲裁 | `{chatScreenshots[]≤5, description≤500}` |
| GET | `/orders/{orderNo}/delivery` | 🔒 | 交割方式/代交接资格 | `{method, canRelay, counterpartyRegion, fee:100}` |
| POST | `/orders/{orderNo}/relay/apply` | 🔑 | 申请平台代交接（支付¥100+发起） | 出：`{relayId, status:"待对方确认"}` |
| POST | `/relay/{relayId}/agree` | 🔒 | 对方确认代交接 | `{agree}` |
| GET | `/orders/{orderNo}/relay` | 🔒 | 代交接进度（4步） | `{steps[], relayStatus, specialist, feePaid}` |
| GET | `/orders/{orderNo}/relay/inspection` | 🔒 | 核验报告 | `{photos[], 成色/克重/品牌, 结论}` |

**订单状态机（核心）：**
```
发布 → selling(销售中, 母单16位)
selling --买家锁价--> 生成子单 locked_pending(锁价待处理, 18位) [4h倒计时]
selling --卖家取消(剩余库存)--> cancelled(剩余部分)
locked_pending --双方都确认完成--> completed(已完成)
locked_pending --任一方申请仲裁--> arbitrating(仲裁中, 暂停倒计时, 平台2h致电)
locked_pending --申请代交接+对方同意--> relay_inspecting(代交接·核验中, 暂停倒计时)
relay_inspecting --验货通过+买家打款--> completed(并释放双方保证金)
locked_pending --倒计时归零--> 🔶 arbitrating 或 defaulted (待业务规则定)
```

> **系统定时任务（非用户触发）**：4h 倒计时归零自动处理；🔶「24h 双方未确认是否自动判完成」待定；代交接完成释放保证金。

# 第四部分：实时与异步专题

## 4.1 实时行情（WebSocket）
- 小程序用 `wx.connectSocket` 建立**单条** WS 连接到 `/ws/market`。
- 客户端订阅：`{ "action":"sub", "metals":["gold"] }`；服务端推 `PriceSnapshot` 增量（含大盘价、涨跌、销售/回购价、报价时间）。
- **必须处理**：心跳（ping/pong）、断线重连（指数退避）、小程序切后台/锁屏断连后重连、弱网降级为轮询 `GET /market/quote`。
- 列表页价格与大盘卡共用同一行情源，避免多连接。
- 🔶推送频率、增量/全量帧格式、是否按品类分频道 → 待定项 C。

## 4.2 锁价「处理中」异步
- `POST /lock/orders` 立即返回 `processing`，前端进入「处理中·缓冲」屏。
- 结果获取二选一（🔶待定项 D）：**A. 轮询** `GET /lock/orders/{id}`（1–2s 间隔，设超时）；**B. WS 推送**结果。
- 提交携带 `snapshotVersion`（价格时效）+ `Idempotency-Key`（防缓冲期重复提交）。

## 4.3 倒计时
- 后端返回 `lockDeadline`(绝对时间) + `countdownRemaining`(秒，可暂停)；**前端基于服务器时间倒计时**，避免本地时钟漂移；仲裁/代交接时后端将倒计时置「暂停」。

## 4.4 支付与资金流
- 保证金充值：`POST /margin/recharge` → 返回 `payParams` → `wx.requestPayment` → 微信回调 `/pay/wechat/callback`（服务器对服务器）→ 余额到账。
- 平台代交接 ¥100：同上支付通道。
- 代交接货款：平台「代收代付」（仅代交接路径涉及平台资金流；自提路径平台不经手货款）。
- 退款：后端发起原路退款，T+1。

# 第六部分：待确定技术方案（Open Decisions）

> 下列为需**产品 / 后端 / 前端共同拍板**的事项，按主题归类。括号内为建议默认值。

## A. 前端技术选型
- **A1** 框架：原生小程序+TS（推荐，微信端、审核稳）/ Taro(React) / uni-app(Vue)？是否未来需支付宝小程序/H5/App 多端？
- **A2** UI：自建组件库（设计稿已有令牌）+ 是否引入 Vant Weapp？
- **A3** 状态管理与请求库选型。

## B. 鉴权与会话
- **B1** 游客态实现：匿名放行 还是 下发游客 token？
- **B2** accessToken / refreshToken 有效期；多端登录与挤下线策略。
- **B3** 联系方式（手机/微信）是否用微信 `getPhoneNumber` 预填，还是纯手填。
- **B4** 静默校验：资质标志随 `/me/profile` 一并下发，还是独立 `/me/eligibility`？

## C. 实时行情机制
- **C1** WebSocket 还是轮询？若 WS：帧格式（增量/全量）、推送频率、是否按品类分频道、鉴权方式。
- **C2** 喂价源与更新频率（金价/银价/铂价）、休市处理。
- **C3** 列表项实时价是否也走 WS（避免每行轮询）。
- **C4** 销售价/回购价/溢价的计算口径与来源。

## D. 锁价并发与时效（最关键）
- **D1** 价格快照有效期（建议 5–15s）；过期后是否允许「按新价重试」。
- **D2**「处理中」缓冲：并发抢锁的**仲裁机制**（先到先得 / 排队 / 抢占窗口）与超时时长。
- **D3** 结果获取：轮询 还是 WS 推送；轮询间隔与超时。
- **D4** 购买上限（等级+保证金）超限：**软约束**（仅提示）还是**提交时硬拦截**？（设计稿为软约束，但金额安全建议后端硬校验）
- **D5** 幂等键生成与有效期、重复提交返回策略。

## E. 金额精度与计算口径
- **E1** 金额传输：**分整数** 还是 **字符串小数**？（建议分整数）
- **E2** 单价小数位（行情 2 位 / 发布溢价 3 位）统一规则；四舍五入策略。
- **E3** 现金价/转账价的**点差来源**：设计稿现金=大盘+2.5、转账=大盘+3.5，是卖家设定的溢价还是平台统一？
- **E4** 百分比传字符串还是数值。

## F. 支付与资金
- **F1** 微信支付商户号与回调地址；充值/退款/代交接服务费支付。
- **F2** 保证金**冻结/解冻/扣罚**的精确时机：锁价时冻结？完成时解冻？违约时扣？（设计稿仅明确「完成即释放」）
- **F3** 平台代交接「代收代付」资金流与对账。
- **F4** 退款 T+1 原路的实现与失败处理。

## G. 业务规则待拍板（设计稿明确标「由业务规则定」）
- **G1** 锁价 4h 倒计时归零 → **自动转仲裁** 还是 **自动判违约**？（或并存）
- **G2** 双方确认完成：是否启用「24h 无确认自动判完成」？时长？
- **G3** 平台代交接：付费方默认发起方、可否协商分摊；是否必须对方同意。
- **G4** 仲裁是否需要独立板块/状态标签（当前订单仅 3 个 tab，无「仲裁中」板块）。
- **G5** 购买上限 / 可发布上限的**精确公式** `f(level, deposit)`。
- **G6** 发布溢价的上下限范围。
- **G7** 品类（category）完整枚举与各金属适配（板料/金条/银条/铂条/旧料…）。
- **G8** 旧料「折后实重」的录入与折算规则。

## H. 实名 / KYC
- **H1** OCR 提供方（微信 OCR 插件 / 第三方）。
- **H2** 是否需人脸核身（需资质）；与公安二要素/三要素比对。
- **H3** 证件照存储与脱敏、合规留存期限。

## I. 合规与审核（硬前置）
- **I1** 小程序主体与**类目资质**（贵金属/金融交易类目审核要求）。
- **I2** ICP 备案 + 小程序备案。
- **I3** 用户协议/隐私政策/交易规则/违约规则文案（法务）。
- **I4** 《买卖双方违约交易细则》《合规须知 5 条》落地文案与展示位。

## J. 消息推送
- **J1** 微信**订阅消息**模板（金价提醒、锁价结果、对方确认、代交接进度、违约通知、平台致电）；一次性订阅授权流程。
- **J2** 短信通道（金价提醒/重要通知）。
- **J3** 角标（订单未完成数）下发方式：轮询 `/orders/badge` 还是推送。

## K. 数据与运维
- **K1** 埋点方案与关键转化漏斗（浏览→锁价→完成）。
- **K2** 错误码最终码值与多语言（暂中文）。
- **K3** 接口文档载体（建议 **Apifox**：文档+Mock+联调一体），前端据此生成 TS 类型并先用 Mock 并行开发。
- **K4** 分包策略与主包体积（主包=行情/首页，分包=交易/我的/订单/发布）。
