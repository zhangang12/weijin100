上架可行性、资质、人脸核验与支付通道调研

针对客户提出的 5 项待调研事项，逐项核实当前（2026 年）政策与技术现状，并突出最需要优先处置的监管风险。

版本 v1.0 日期 2026-07-13 范围 贵金属（黄金）C2C 撮合 + 线下实物交割 性质 内部调研 · 非法律意见

▲核心结论：最大的不确定性不在「能不能上架」，而在「业务模式本身是否合规」

深圳市金融委办公室 2026-02-12 已公开发文，明确禁止一类黄金交易行为——其原文表述与「微金100」的「保证金 + 锁价」机制字面高度重合：

「……允许客户缴纳一定金额保证金、锁定一定重量黄金的价格，后随金价波动按价差结算……」属违规开展黄金预定价交易；并禁止「通过微信群或小程序、APP、网站等互联网平台」开展，禁止「开发、兜售非法黄金交易软件、APP 或小程序」，禁止「为其提供信息服务和技术支持」。

可辩护的关键区别（务必据此设计与自证）：被禁的是「按价差现金结算」的对赌/预定价模式；而微金100 的定位是「锁价 → 4 小时内线下实名实物交割 → 双方确认完成」的实物现货撮合，保证金为履约保证金而非杠杆。二者法律性质不同。但——「锁价 / 保证金锁定价格」的措辞与形式正是监管重点盯防对象，且水贝就在深圳辖区。

建议：在投入上架之前，先做一次法律合规评估（金融/互联网合规律师），并据此调整机制与措辞（见文末「建议下一步」）。上架的技术问题（备案、类目、人脸、支付）都在此之后、且都可解。

01上架可行性与周期

客户："有条件 app 上架审核的是不是"、"周期在 1-2 周吧"、"重点是上架，这个不可控"

分两条线：小程序、APP。两者共同的硬前提是「先备案」——自 2023-09 起，微信小程序、以及安卓各大商店 + 苹果中国区 App Store，均要求先完成 ICP 备案（拿到备案号）才能上架，未备案直接驳回。

小程序

  • 备案：2023-09-01 起强制。非经营性 ICP 备案免费，周期约 15–20 个工作日。
  • 类目资质：小程序功能必须与所选服务类目一致，贵金属/黄金属敏感类目，需提交与主体经营范围匹配的资质，类目审核一般 1–7 天。
  • 现实通过率：检索到的行业数据显示，特殊行业（含金融/贵金属）App/小程序备案通过率显著偏低（约 42%，普通应用约 75%）；常见驳回：主体与资质不一致、经营范围不匹配、资质过期/模糊。

APP

  • 备案：同样强制 ICP 备案。非经营性备案 15–20 工作日；经营性需办 ICP 许可证（30–60 工作日，见 02）。另需软件著作权。
  • 金融类特殊要求：各安卓商店（华为/小米/OPPO/vivo/应用宝）对金融类要求差异大，部分品类（证券/基金/彩票等）需相应牌照或授权文件；黄金/贵金属类审核趋严。

结论

客户预估的「1–2 周」过于乐观。仅备案就要 15–20 工作日起，叠加类目/资质审核与贵金属类目的低通过率、反复驳回，实际是数周到数月、且不可控——客户"重点是上架、这个不可控"的判断是对的。技术侧「做出来」确实不是瓶颈。

02上架资质要求：企业主体够不够？

客户："app 上架资质要求应该没有小程序高，他只需要企业主体就好了吧"、"我看看把营业范围改一下"

企业主体是「必要但不充分」条件。营业执照（企业主体）只是入场券，贵金属交易类应用还叠加两层：备案层 + 行业金融监管层。

企业主体
必须为企业（非个人），营业执照经营范围需覆盖对应业务（客户已想到"改营业范围"——方向对，但只是第一步）。
软件著作权
APP 上架各商店普遍要求。
ICP 备案 vs 许可证
备案非经营性（免费、无注册资本要求）；许可证经营性（增值电信业务经营许可证）——注册资本 ≥ 100 万元、近一月 3 名社保、需年审。撮合平台向用户收费/抽佣，通常被认定为经营性。
行业金融监管
央行等规定：金融机构未按规定向金融监管部门备案或获批，不得开展黄金业务。贵金属交易触碰此线。

结论

"只要企业主体"是不准确的。改营业范围是必要动作，但真正的门槛是：① 大概率需 ICP 许可证（100 万注册资本级门槛）；② 贵金属经营可能需行业备案/审批并履行反洗钱义务（见文末新规）。这几项决定了"能不能合法开展"，而非仅"能不能上架"。

03人脸识别 / 核验接入

客户："小程序是不是不能调用人脸识别，app 可以调用人脸是吧"、"人脸可以尝试接入吗"

纠正一个误解：小程序也能做人脸核验，不是"不能调用"。只是小程序走的是"审核制"，APP 更自由。人脸/实名一律接第三方服务（不自研），这一项是本次调研里风险最低、最可落地的。

小程序

  • 微信原生：wx.startFacialRecognitionVerify（人脸核身接口），在 mp 后台「行业能力 → 人脸核身」开通。
  • 门槛（关键）：主体 + 类目需在限定范围内且与业务场景一致；业务须是"国家法规规定需实名办理"的类型；不接受资质授权（子公司不能借用集团/关联公司资质）；审核约 1–3 工作日。
  • 第三方替代：腾讯云慧眼提供"微信小程序 SDK"人脸核身，同样需企业实名 + 符合行业资质。

APP

  • 可接任意第三方：腾讯云慧眼人脸核身、阿里云实人认证等；企业实名开通、创建密钥即可，接入更灵活。
  • 可与公安权威库比对（活体 + 身份二要素/三要素核验），按次计费。

✓结论

人脸核验技术上完全可接，小程序、APP 都能做，第三方（腾讯云/阿里云）方案成熟、按次计费。小程序需先过类目资质审核（与 01/02 同一前提——类目通过是钥匙）；APP 侧更自由。此项本身不是障碍，其瓶颈仍回到"类目/资质是否通过"这个上游问题。

04支付通道费用（进 / 退款 / 金额调拨）

客户："支付端口的进退款项，因为我们有金额调拨的发生，这些是否会产生通道费用"
进账费率
微信支付商户默认 0.6%（千分之六）；通过服务商可申请降至 0.2%（千分之二）。每一笔用户付款（保证金充值、代交接 ¥100 等）按此收取。
退款
退款时不再另收交易手续费；但原付款时已扣的手续费不退还。含义：频繁"充值→退款"会累积沉没手续费（每进一次就损耗一次 0.6%/0.2%）。
提现
结算到商户号后提现到银行卡，平台按经营状况可能有提现周期/规则；具体费率以商户协议为准。
金额调拨 / 资金归集 高压线
若平台做"代收代付、资金归集、二次分账"（如保证金池、买卖方之间划转、代交接代收代付），涉及特约商户 / 分账能力，且形成资金池是无金融牌照不可碰的红线。保证金/代收代付资金应走银行存管 / 持牌支付机构分账，不能沉淀在自有账户。

结论

常规进账 0.6%（可谈 0.2%）、退款不重复收但已扣不退——这部分可控、可预算。真正要当心的是"金额调拨"背后的资金合规：保证金冻结/解冻/扣罚赔付、代交接代收代付，一旦形成"资金池 / 平台代持用户资金"，就触碰金融牌照红线。建议资金侧采用持牌支付机构的分账/存管方案，把钱的通道与合规交给持牌方，平台只做撮合与记账。

05APP 打包与上架

客户："做出来不要什么时间，重点是上架，这个不可控"、"可以出 APK 和 iOS 的包"、"app 最后都是要做的，想着做先一版"
  • 出包：技术上不难。当前工程是微信小程序，转 APP 需重做或用跨端方案（uni-app / 壳套 WebView）——"做出来"确实不是瓶颈。
  • 上架：安卓需分别过华为/小米/OPPO/vivo/应用宝等各商店审核；iOS 过 App Store。全部要 ICP 备案；金融/贵金属类审核严、周期长、结果不可控。
  • APP vs 小程序的取舍：APP 在人脸/硬件能力、审核弹性上更自由；但上架资质与备案门槛并不比小程序低（同样卡在贵金属类目 + ICP）。客户"app 问题不是很大、没有小程序复杂"的判断，仅在"功能实现/人脸接入"层面成立，在"上架合规"层面并不成立。

结论

"先做一版 APP"可行且能推进产品；但不要期待 APP 能绕开小程序遇到的合规/上架门槛——二者上游是同一个问题（贵金属类目 + 备案 + 资质）。APP 的价值在于人脸/功能更自由 + 可脱离微信生态，而非"更好上架"。

补充2025 年反洗钱新规（务必并入合规设计）

央行《贵金属和宝石从业机构反洗钱和反恐怖融资管理办法》（银发〔2025〕124 号）已将贵金属从业机构纳入反洗钱监管。这意味着即便业务模式合规，平台也须履行：

  • 客户身份识别（KYC）：实名 + 人脸核验正是为此——本项目的实名/人脸设计方向正确、且是合规刚需而非可选项。
  • 大额交易 / 可疑交易报告：达到额度或触发可疑特征需上报。
  • 客户身份资料与交易记录留存。

好的一面：项目已内置实名认证、保证金、交易记录、违约/仲裁等机制，与反洗钱的"可追溯、可核验"要求方向一致，是合规资产而非负担。

✓建议下一步（按优先级）

  1. 先做法律合规评估（最高优先）：请金融/互联网合规律师就"锁价 + 保证金 + 4h 实物交割"模式出具意见，明确它与被禁的"预定价/价差结算"如何切割、需要哪些自证材料。这决定整件事能不能做，其余都在此之后。
  2. 据评估调整机制与措辞：弱化"锁价 / 保证金锁定价格"这类敏感表述（可改为"预订下单 / 库存锁定 / 履约保证金"）；强化"实物现货、线下实名交割、非价差结算、非杠杆、非延期"的产品定性与流程留痕。
  3. 并行准备资质：企业主体 + 经营范围调整 → ICP 备案 → 视经营性判断是否办 ICP 许可证（100 万注册资本级）→ 贵金属经营/反洗钱相关备案。
  4. 资金侧走持牌方案：保证金/代收代付接持牌支付机构分账/银行存管，不自建资金池。
  5. 人脸/实名接第三方（腾讯云慧眼 / 阿里云实人认证），既满足体验也满足反洗钱 KYC——技术上随时可接，等类目/资质就绪即开通。
  6. 上架预期管理：告知客户周期以"月"计、结果不可控；可先内测/企业分发验证产品，正式上架待合规就绪。
