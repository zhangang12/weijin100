# 微金100 小程序 · 后端接口对接 · 待确认清单

> **配套文档**：《微金100小程序_接口文档方案与技术待定项_v0.1.md》（接口/字段/错误码细节）、《微金100_业务规则确认表.html》（业务规则，由业务方确认，本清单不重复）。
> **怎么用**：请逐条在「**后端确认**」列填写结论（采用建议 / 调整为…）。带 ⚠️ 的为阻塞项，建议优先。
> **前端已定前提**：原生微信小程序 + TypeScript + 自建组件库；前后端分离；金额/计算以**后端为准**。

---

## 0. 已确定的前端前提（供后端对齐，无需确认）

- 客户端：微信小程序（原生 + TS），单端。
- 登录：`wx.login` → JWT；游客可浏览行情，操作（锁价/发布/充值）需登录。
- 手机号：采用**微信一键授权 `getPhoneNumber`**（→ 需后端提供手机号解密接口，见 §1）。
- 实时行情、锁价处理中等需要**服务端推送或轮询**支持（见 §3、§4）。

## 1. 登录与鉴权 ⚠️

| 待确认点 | 前端建议 | 后端确认（请填） |
|---|---|---|
| 登录流程：`wx.login` 拿 `code` → 后端 `jscode2session` 换 openid/unionid/session_key → 签发 JWT | 采用此流程；返回 `accessToken + refreshToken + expiresIn + user` | |
| 游客态实现 | 浏览类接口**匿名放行**（无需 token）；或下发游客 token | |
| Token 有效期与续期 | accessToken 短期（如 2h）+ refreshToken 长期（如 30d）；401 用 refresh 续，失败重登 | |
| 多端登录/挤下线策略 | 由后端定（小程序单端，建议允许并存） | |

| 资质标志下发（静默校验依据） | 提供 `GET /me/eligibility?action=lock|publish` → `{realName, contact, marginOk, functionStatus, maxQty}`；或随 `/me/profile` 返回 | |
| 请求头 | `Authorization: Bearer`、`Idempotency-Key`（写操作）、`X-Request-Id` | |

## 4. 锁价：快照、并发、幂等、时效 ⚠️（核心）

| 待确认点 | 前端建议 | 后端确认（请填） |
|---|---|---|
| 价格快照机制 | 报价返回 `snapshotVersion`；提交锁价携带该版本，后端校验是否过期 | |
| 快照有效期 | 建议 5–15s（业务侧见确认表 A3） | |
| 提交锁价接口 | `POST /lock/orders`，带 `Idempotency-Key`；立即返回 `status:"processing"` | |
| 处理中并发抢锁 | 服务端做并发仲裁（先到先得/排队，业务侧见确认表 A4），前端轮询/等待结果 | |
| 结果获取 | **轮询 `GET /lock/orders/{id}`**（1–2s，设超时）或 WS 推送，二选一请定 | |
| 幂等键 | 写操作（锁价/发布/充值/确认完成）均带 `Idempotency-Key`，有效期与重复返回策略由后端定 | |
| 提交时硬校验 | 库存、保证金/上限、联系方式齐全均**后端二次校验**（前端跳转仅体验优化） | |
| 失败原因 | 失败返回 `failReason`/错误码（被抢锁 `LOCK.GRABBED`、过期 `LOCK.QUOTE_EXPIRED`、库存不足等） | |

## 5. 金额精度与计算口径 ⚠️

| 待确认点 | 前端建议 | 后端确认（请填） |
|---|---|---|
| 金额传输 | **以「分」整数**传输（避免 float）；或统一字符串小数，二选一请定 | |
| 单价小数位 | 行情/成交价 2 位；发布溢价/固定价 3 位；四舍五入规则由后端定 | |
| 百分比格式 | 字符串 `-0.49%` 或数值，二选一请定 | |
| 计算口径 | 克重×单价、现金/转账合计、限额、溢价、保证金额度**一律后端计算**，前端仅展示/试算 | |
| 现金/转账点差 | 数值来源（卖家溢价 vs 平台统一）业务侧见确认表 A5；接口需分别返回现金价/转账价 | |
