## 1. 技术栈（定稿）

| 层 | 选型 | 说明 |
|---|---|---|
| 前端 | 微信小程序 **原生 + TypeScript** + 自建组件库 | 已建 21 页，`tsc` strict 0 报错 |
| 后端框架 | **NestJS + TypeScript** | 模块化/DI/管道校验/守卫 |
| 数据库 | **PostgreSQL + Prisma** | 资金/订单强一致；Prisma 类型安全 + 迁移 |
| 缓存/锁/队列 | **Redis + BullMQ** | 行情缓存、分布式锁(并发抢锁)、幂等、延时任务 |
| 实时 | **WebSocket** | 行情推送（脉动已接）、订单/锁价状态推送 |
| 鉴权 | **JWT(access+refresh) + 微信 `jscode2session`** | 游客可浏览，操作时校验 |
| 支付 | **微信支付 V3** | 保证金充值、代交接服务费 |
| 对象存储 | **阿里云 OSS**（STS 直传） | 证件 / 商品图 / 仲裁截图 |
| 实名 | 第三方 OCR + 人脸核身 | 需资质 |
| 短信 | 阿里云短信 | 金价提醒 / 通知 |
| 部署 | **阿里云** ECS + RDS(PG) + Redis + OSS + SLB；Docker + Nginx(HTTPS) | |
| 行情源 | **脉动 PulseData**（已接，黄金=RT_AU 融通金） | 正式端口 1008 需授权服务器 IP |
| 协作 | Git(GitHub) + Apifox(OpenAPI) + 本地 docker-compose(pg+redis) | |

## 2. 系统架构

```
微信小程序(原生+TS) ──HTTPS REST──┐
        └────────WSS 行情/状态──┐ │
                                ▼ ▼
                    Nginx(HTTPS) → NestJS 后端
                    ├─ PostgreSQL(RDS)  业务主库（订单/保证金/用户…）
                    ├─ Redis            缓存 / 分布式锁 / 幂等 / BullMQ 延时任务
                    ├─ OSS              证件/图片
                    ├─ 微信开放平台      登录 jscode2session / 微信支付 V3 / 订阅消息
                    ├─ 阿里云短信        金价提醒/通知
                    └─ 脉动 PulseData    行情(HTTP 轮询 + WS)；正式端口需授权服务器 IP
```

## 4. 联调约定（前后端共同遵守）

| 约定 | 规则 |
|---|---|
| 响应信封 | `{ code, bizCode, message, data, requestId }`；`code=0` 成功 |
| 错误码段 | `0` 成功；`1xxx` 鉴权/会话；`2xxx` 参数校验；`3xxx` 业务规则拒绝（含 `bizCode` 如 `MARGIN_NOT_ENOUGH`/`LISTING_SOLD`/`OVER_LIMIT`）；`5xxx` 系统/上游 |
| **金额单位** | **一律「分」整数**（前端展示自行 /100 格式化）；后端为准 |
| 重量单位 | 克；可带小数（Decimal） |
| 价格 | 字符串（如 `"891.00"`），后端格式化；快照为准 |
| 时间 | ISO8601 带时区 `+08:00` |
| 鉴权 | `Authorization: Bearer <jwt>`；游客可读，操作类静默校验 → `me/eligibility` |
| 幂等 | 写操作（下单/支付/锁价）带 `Idempotency-Key`，Redis 去重 |
| 分页 | `?page&pageSize` → `{ list,page,pageSize,total,hasMore }` |
| Mock→真切换 | 小程序 `config/env.ts`：`USE_MOCK=false` + `BASE_URL` 指向后端；后端开发期 mock 接口逐步替换为真逻辑，路径/结构不变 |

> 技术口径（原《后端待确认清单》）现由后端拍定：token 有效期 access 2h / refresh 30d；价格快照有效期 = 倒计时 4h；金额分；统一信封与错误码如上。

## 5. 开发路线图（里程碑）

> 前后端并行，每阶段以「可联调」为完成标准。Sprint 长度约 1–2 周（按团队定）。

| Sprint | 目标 | 后端 | 前端 |
|---|---|---|---|
| **0 工程基建** | 跑起来、能联调 | NestJS 骨架（✅ 进行中）+ Prisma 模型 + 本地 docker(pg/redis) + 行情真链路(✅) + CI(lint/typecheck) | 接通真后端行情、统一 env 切换、联调环境 |
| **1 账户与鉴权** | 进得来、看得到身份 | 微信登录(jscode2session)+JWT+游客态、实名 KYC、我的/级别、保证金账户(充值走支付沙箱)、`me/eligibility` 静默校验 | 登录态、我的/实名/保证金/级别页接真接口 |
| **2 行情与发布** | 看行情、能挂单 | 行情(✅)+订阅提醒+短信、商家发布挂单(校验+合规)、挂单列表/详情 | 首页/行情、发布表单、挂单详情接真接口 |
| **3 锁价核心闭环** | 重头戏 | 价格快照、**并发抢锁(Redis 锁,先到先得 A4)**、保证金冻结(C5)、锁价单→订单、**4h 倒计时(BullMQ)**、双方确认完成、24h 自动完成 | 锁价全流程、订单确认接真接口 |
| **4 交割与异常** | 收尾闭环 | 平台代交接¥100(支付)、仲裁、**违约判定/扣罚/降级/限制**、违约申诉 | 代交接/仲裁/违约/申诉页接真接口 |
| **5 联调·合规·上线** | 上线 | 全链路联调、压测、审计、灰度 | 全量回归、提审、灰度 |

- **资质与备案周期最长**——务必最先启动，否则上线受阻（即使代码就绪也无法发布）。
- **行情正式端口需授权后端服务器公网 IP**（未授权返回 500），上线前必须完成；铂金代码缺失。
- **微信支付资质**审批周期长，影响保证金/支付链路。
- **数据源单点**：建议加缓存 + 降级（脉动断流时用最后快照 + 告警）。
