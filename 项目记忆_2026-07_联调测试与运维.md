## 一、30 秒项目背景

- **产品**：微金100，微信小程序，**贵金属（金/银/铂）C2C 撮合 + 线下实名交割**。卖家发布现货挂单 → 买家锁价（快照价 + 冻结保证金）→ 4h 内线下交割 → 双方确认完成 → 解冻。含保证金、等级佣金、违约申诉、平台代交接、仲裁。
- **技术栈**：
  - 前端 `frontend/`：**原生微信小程序 + TypeScript**，自建组件库，分包（主包=行情，分包=交易/我的/订单/发布）。
  - 后端 `backend/`：**NestJS + TypeScript + PostgreSQL(Prisma) + Redis(BullMQ)**；行情对接脉动 PulseData（黄金 `RT_AU` 融通金）；金额一律「分」(BigInt) 存储。
  - 统一响应信封：成功 `{code:0,bizCode:'OK',data,...}`；业务拒绝 `{code:<3xxx>,bizCode,message,data:null}`（HTTP 200）；鉴权失败 `code:1001`。
- **仓库**：`github.com/zhangang12/weijin100`，主分支 `main`，开发分支 `claude/user-session-loading-3be30w`。

## 三、联调运维手册（可复用操作）

> 以下命令均在**你本机（Mac）**执行；云会话里的差异见 §四·git 推送。

### 1. 启动后端
```bash
cd backend
npm install                       # 首次
# 确保 PostgreSQL 在跑（见 §五·PG 自停）
npx prisma generate               # 类型变更后必跑，否则 tsc 报字段不存在
npx prisma migrate deploy         # 应用迁移
npm run seed                      # 种子：demo_buyer / demo_seller / 3 挂单
npm run start:dev                 # 热重载启动（端口 3100）
```
看到 `微金100 后端已启动 … REST : http://localhost:3100/api/v1` 即成功。
> 环境变量：本机放 `backend/.env`（Prisma 自动加载）；联调基线见 `backend/.env.test`。

### 2. 前端接后端（DevTools）
1. DevTools **打开 `frontend/` 目录**（不是 `frontend/miniprogram/`）。
2. **详情 → 本地设置 → 勾「不校验合法域名…」**。
3. 点「编译」。`config/env.ts` 已配 `USE_MOCK=false`、BASE_URL=`127.0.0.1:3100`、`DEV_LOGIN=true`。
4. **真机调试**：把 env.ts 的 dev BASE_URL 改成电脑局域网 IP（`http://192.168.x.x:3100/api/v1`），手机与电脑同 WiFi，点「真机调试」扫码。

### 3. DEV_LOGIN 联调登录机制
- `DEV_LOGIN=true` 时前端登录发 `code = 'mock:' + DEV_OPENID`；后端 `wechat.service` 见 `mock:` 前缀直接把后缀当 openid 发 JWT，**无需真实 AppID**。
- 种子账号：买家 `mock:demo_buyer`（¥30 万保证金、已实名）、卖家 `mock:demo_seller`。
- **上线前务必**：`DEV_LOGIN=false`、`ENV='prod'`。

### 4. 查看后端请求日志
- 已加中间件，终端实时打印 `GET /api/v1/...`（过滤了 SdkReport）。
- 更快的方式：DevTools **Network** → 点请求 → **Response** 看返回 JSON 的 `code/message`。

### 5. 跑测试（`backend/` 下）
```bash
npm run typecheck     # 类型检查（前后端都有）
npm run test:unit     # 单元测试 31 例（无需 DB/服务）
npm run seed && npm run start &   # 集成前先起服务
npm run test:api      # API 集成 63 例（需服务 + 真库）
```
前端：`cd frontend && npm ci && npm run typecheck`。

### 6. 重启后端 / 重启 PostgreSQL
```bash
# 重启后端：终端按 Control+C（Mac 也是 Ctrl 不是 Cmd），再 npm run start:dev
# 重启 PG（连接被拒 P1001 时）：
pg_ctlcluster 16 main start       # Linux；Mac 用 brew services 或 Postgres.app
```

## 五、git 推送（环境差异，重要）

- **你本机（Mac）**：macOS 钥匙串已有 GitHub 凭证，`git push origin <branch>` 正常；`.github/workflows/` 也能推（凭证有 workflow scope）。
- **本云会话**：git 走本地代理会 403/401。改用 `~/.git-credentials` 里的 PAT 直推：
  ```bash
  git push "https://zhangang12:<PAT>@github.com/zhangang12/weijin100.git" HEAD:<branch>
  git fetch "https://…@github.com/…" <branch>:refs/remotes/origin/<branch>   # 同步 tracking ref
  ```
  - PAT 本身存于本地凭证文件（**不要写进任何提交/文档**）。
  - **限制**：该 PAT 缺 `workflow` scope → 任何新增/改动 `.github/workflows/` 的推送都会被 GitHub 拒绝。故 `ci.yml` 需由你在 Mac 上添加。
