买家锁价（点击锁价 → 锁单交割）

买家在「行情」列表点击 锁价 按钮进入。整体风格沿用货品发布稿的 section header + card + field 排版（高端财金 App 范式）；主报价模仿首页奶油金渐变卡。
右上角双开关：设计说明（S 键）／批注模式（A 键，点屏添加意见 → 导出 Markdown）。

· 平台权限模型 · 游客可看 + 会员可操作

本平台是「查询 + 交易」一体系统。游客（无需登录/注册）即可免费浏览展示信息。仅当用户主动点击操作按钮（锁价 / 发布货品）时，才触发静默校验，直接路由跳转「我的」补全必填项后继续。设计不在入口处强制实名，校验无弹窗。

动作游客 · 免登录会员 · 已补全
浏览所有展示信息（行情 / 商家 / 商品 / 金价 / 锁价主页含数量试算）✓ 免费✓
主动操作（锁价 / 发布货品）→ 直跳「我的」补全资料✓

跳转规则（无弹窗）：
· 锁价（买家）：点「确认锁价」时静默校验联系方式（手机 / 微信）→ 未填则直接路由跳转「我的 - 我的信息」补全 → 返回保留数量 → 再点直接进屏 ④。
· 购买上限：基于等级 + 保证金动态计算，超限时数量下方提示卡显示「补足保证金 →」链接路由跳「我的 - 保证金充值」（软约束，不在 CTA 点击时硬阻拦）。
· 发布货品（卖家）：校验更严格，需补全实名 + 联系方式 + 保证金（详见货品发布稿）。
所有补全页 UI（我的信息 / 保证金充值 等）均不在本稿绘制，属「我的」模块单独设计。

批注 · 锁价主页 · 整出全量

  • 1商品图片轮播：右上角性质标签（板料 / 旧料 / 现货）+ 检测证书徽，沿用首页 chip 风格。
  • 2slim 商家行（取代原大卡）：头像 + 用户名 + 紫色 L9 徽章 + 商品标记字 + 成交量胶囊。一行扼要。
  • 3实时报价大卡（仿首页金价卡）：奶油渐变顶部 + 大盘价 26px 大字 + 涨跌色 + 迷你折线；下半部白底现金/转账两价并列；红/蓝迷你 chip 区分支付方式。
  • 4剩余库存进度条整合进同一张报价卡底部，减少独立小卡。
  • 5数量与订单合计合并为同一张卡（应客户要求）：上半为数量（场景① 整出全量带锁图标，22px 大字 10,000.00 g，不可调），同卡下半直接显示现金/转账总价 + 公式。调整数量时总价始终在视野内，避免滚动看不到总额而误操作。
  • 6等级 + 保证金 + 可购买上限 提示（数量正下方）：
    · 左侧 chip 显示「我的级别 L1-L9」（紫色，对应首页等级体系）+「保证金 ¥XX,XXX」（金色）；
    · 中间行显示「可购买总数量 1 ~ X g」（根据等级×保证金动态计算）+ 当前数量超限时红字提示；
    · 底部金色虚线链接「补足保证金 →」路由跳转「我的 - 保证金充值」页（不弹窗，inline 跳转），补完返回本页保留数量。
    · 跳转目标 UI 不在本稿绘制，属「我的」模块。
    · demo 为 L1 + ¥0 + 可购 0g + 当前 10,000g 超出的受限态；不同等级和保证金结果不同（公式由业务规则定义）。
  • 7「确认锁价」点击后系统静默校验联系方式（手机 / 微信）：
    · 已填写 → 弹出锁价订单详情（屏 ④）；
    · 未填写 → 直接路由跳转到「我的 - 我的信息」（不弹窗、不浮层），补完返回本页保留已填数量，再点「确认锁价」直接进屏 ④。
    · 锁价仅校验联系方式（卖家需要电话/微信对接交割）；保证金通过数量上限提示（pin 6）做软约束而不在此校验。
    · 我的信息录入 UI 不在本稿绘制，属「我的」模块，留待该模块单独设计。
